# Gomobile Interoperability Rules

The bridge between Go and Kotlin is generated by `gomobile bind`. It has strict limitations on what types can cross the language boundary.

## Supported Types

Only the following types can be exported from Go and used in Kotlin:

1. **Primitives:** `int`, `int64`, `float64`, `bool`, `string`.
2. **Byte Slices:** `[]byte` (Maps to `byte[]` in Java/Kotlin).
3. **Structs:** Exported Go structs map to Java classes.
4. **Interfaces:** Exported Go interfaces map to Java interfaces. Kotlin can implement these interfaces and pass them to Go.

## Unsupported Types

**DO NOT** try to export functions or structs that use:
- `uint`, `uint64` (Use `int` or `int64` instead).
- Maps (e.g., `map[string]int`). Instead, serialize to a JSON `[]byte` or `string` in Go, and parse it in Kotlin.
- Slices of any type other than `byte` (e.g., `[]string` is NOT supported). Pass a comma-separated `string` or JSON instead.
- Pointers to basic types (e.g., `*string`).
- Functions as arguments (use an Interface with a single method instead).

## Best Practices for BlockAds

### 1. Passing Complex Data
If you need to pass a list or an object from Go to Kotlin, encode it as JSON in Go and return a `string` or `[]byte`.
```go
// Go
func (e *Engine) GetStatsJSON() string {
    stats := Stats{ Total: 10, Blocked: 2 }
    b, _ := json.Marshal(stats)
    return string(b)
}
```

### 2. Callbacks (Kotlin -> Go)
To have Go call Kotlin code, define an interface in Go, implement it in Kotlin, and pass the instance to Go.

```go
// Go
type LogCallback interface {
    OnDNSQuery(domain string, blocked bool)
}

func (e *Engine) SetLogCallback(cb LogCallback) {
    e.logCallback = cb
}
```

```kotlin
// Kotlin
class MyLogCallback : tunnel.LogCallback {
    override fun onDNSQuery(domain: String, blocked: Boolean) {
        // Handle in Kotlin
    }
}

engine.setLogCallback(MyLogCallback())
```

### 3. Page Size Limitations (Android 15)
Android 15 requires apps to support 16KB page sizes. This is handled in `scripts/build_tunnel.sh` via the linker flag: `-extldflags=-Wl,-z,max-page-size=16384`. Never remove this flag when rebuilding the tunnel.
